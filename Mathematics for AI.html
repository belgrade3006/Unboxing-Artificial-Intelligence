Vectors and Vector Operations

Definition 1 (Vector). We know of vector as a quantity having both magnitude and direction. Now if we define vector as a 1D array of data points, it hits us hard. The subtle difference is the space we are working in. Physicists work in physical space confined to 3 dimensions, while theoreticians work in data space, which can have any number of dimensions. Hence defining a vector v as an ordered collection of n real numbers 𝒗∈ℝ\mathbf{v} \in \mathbb{R} in a 1D matrix: 𝒗=[v1v2⋮vn]\mathbf{v} = \begin{bmatrix} v_1 \\ v_2 \\ \vdots \\ v_n \end{bmatrix} actually represents a vector in data space made up of n dimensions where v1 is the component of the vector along one particular axis, v2, along another axis and so on. These axes may or may not be orthogonal to one another. A 2D array array might be known as a matrix but a 1D array is not conventionally known as a matrix. It is known as an array.

A question might arise as to why we work with vectors, represented by 1D arrays in data space. Matrices function like a container which encloses all data entries. It allows us to group an entire data set into a single mathematical enitity. Operating on seperate entities using a for loop is incredibly slow. Parallel processing is a computer architecture, which operates on all the data entries of a matrix in one go, without using do loops, thereby making the process incredibly efficient.

Example 1 (Chemical Concentration Vector). A vector representing concentrations of 4 chemicals in a solution: 𝒄=[0.51.20.80.3](in mol/L)\mathbf{c} = \begin{bmatrix} 0.5 \\ 1.2 \\ 0.8 \\ 0.3 \end{bmatrix} \quad \text{(in mol/L)}

  • Addition: 𝒖+𝒗=[u1+v1,u2+v2,…,un+vn]T\mathbf{u} + \mathbf{v} = [u_1+v_1, u_2+v_2, \dots, u_n+v_n]^T

  • Scalar Multiplication: α𝒗=[αv1,αv2,…,αvn]T\alpha\mathbf{v} = [\alpha v_1, \alpha v_2, \dots, \alpha v_n]^T

  • Dot Product: 𝒖⋅𝒗=∑i=1nuivi\mathbf{u} \cdot \mathbf{v} = \sum_{i=1}^n u_i v_i

  • Norm (Length): ∥𝒗∥=𝒗⋅𝒗=∑i=1nvi2\|\mathbf{v}\| = \sqrt{\mathbf{v} \cdot \mathbf{v}} = \sqrt{\sum_{i=1}^n v_i^2}

For a square matrix 𝑨∈ℝn×n\mathbf{A} \in \mathbb{R}^{n \times n}, if there exists a scalar λ\lambda and non-zero vector 𝒗\mathbf{v} such that: 𝑨𝒗=λ𝒗\mathbf{Av} = \lambda\mathbf{v} then λ\lambda is an eigenvalue and 𝒗\mathbf{v} is an eigenvector of 𝑨\mathbf{A}.

Functions and Derivatives

Definition 4 (Derivative). The derivative of a function f(x)f(x) at point xx measures the instantaneous rate of change: f′(x)=limh→0f(x+h)−f(x)hf'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h} This is known as the forward difference formula. It depends on the value of the function at x and in its neighbourhood.

Example 3 (Reaction Rate). The rate of a chemical reaction as a function of temperature TT: r(T)=ke−Ea/(RT)r(T) = k e^{-E_a/(RT)} EaE_a is the Activation Energy, k is the rate constant for the reaction and R is the universal gas constant. EaE_a and T both can change simultaneously but since it has been already mentioned that rate of the reaction is sheerly a function of temperature, EaE_a is assumed to be a constant. The derivative r′(T)r'(T) tells us how sensitive the reaction rate is to temperature changes or the rate of change of r(T) with temperature.

Example 4 (Optimization of a Simple Cost Function). Suppose a simple machine learning model’s error is given by E(w)=3w2−12w+15E(w) = 3w^2 - 12w + 15. To find the optimal weight ww that minimizes the error, we take the derivative and set it to zero: E′(w)=6w−12=0⟹w=2E'(w) = 6w - 12 = 0 \implies w = 2 This shows the optimal weight is 2.

Partial Derivatives and Gradients

Definition 5 (Partial Derivative). For a function f(x1,x2,…,xn)f(x_1, x_2, \dots, x_n), the partial derivative with respect to xix_i is: ∂f∂xi=limh→0f(x1,…,xi+h,…,xn)−f(x1,…,xn)h\frac{\partial f}{\partial x_i} = \lim_{h \to 0} \frac{f(x_1,\dots,x_i+h,\dots,x_n) - f(x_1,\dots,x_n)}{h}

Definition 6 (Gradient). The gradient of f:ℝn→ℝf: \mathbb{R}^n \to \mathbb{R} is a vector of all partial derivatives: ∇f(𝒙)=[∂f∂x1∂f∂x2⋮∂f∂xn]\nabla f(\mathbf{x}) = \begin{bmatrix} \frac{\partial f}{\partial x_1} \\ \frac{\partial f}{\partial x_2} \\ \vdots \\ \frac{\partial f}{\partial x_n} \end{bmatrix} The gradient points in the direction of steepest ascent.

The individual elements or dimensions of the gradient vector actually represent how the function changes if only that dimension changes, keeping all other dimensions constant. The gradient vector as an array represents how the function varies if all the dimensions change. This change is just a vector sum of the changes due to single dimensions.

Example 5 (Energy Surface Gradient). In molecular modeling, the gradient of the potential energy surface indicates the direction atoms will move to minimize energy.

Example 6 (Calculating a Gradient). Let f(x,y)=x2y+sin⁡(x)f(x, y) = x^2y + \sin(x). The partial derivatives are: ∂f∂x=2xy+cos⁡(x)\frac{\partial f}{\partial x} = 2xy + \cos(x) ∂f∂y=x2\frac{\partial f}{\partial y} = x^2 The gradient vector is ∇f(x,y)=[2xy+cos⁡(x)x2]\nabla f(x,y) = \begin{bmatrix} 2xy + \cos(x) \\ x^2 \end{bmatrix}. Evaluating this at (0,1)(0, 1) gives ∇f(0,1)=[10]\nabla f(0,1) = \begin{bmatrix} 1 \\ 0 \end{bmatrix}. If the gradient vector is 0, it indicates the slope of the function is flat, or it has reached either a maxima, a minima or a saddle point.

The Hessian Matrix (Second-Order Derivatives)

Definition 7 (Hessian Matrix). The Hessian matrix 𝑯\mathbf{H} is a square matrix of second-order partial derivatives, critical for understanding the curvature of loss landscapes: 𝑯i,j=∂2f∂xi∂xj\mathbf{H}_{i,j} = \frac{\partial^2 f}{\partial x_i \partial x_j}

Example 7 (Hessian Calculation). For f(x,y)=x2+3xy+2y2f(x,y) = x^2 + 3xy + 2y^2:

  • First derivatives: ∂f∂x=2x+3y\frac{\partial f}{\partial x} = 2x + 3y, ∂f∂y=3x+4y\frac{\partial f}{\partial y} = 3x + 4y

  • Second derivatives: ∂2f∂x2=2\frac{\partial^2 f}{\partial x^2} = 2, ∂2f∂y2=4\frac{\partial^2 f}{\partial y^2} = 4, ∂2f∂x∂y=3\frac{\partial^2 f}{\partial x \partial y} = 3, ∂2f∂y∂x=3\frac{\partial^2 f}{\partial y \partial x} = 3

The Hessian is 𝑯=[2334]\mathbf{H} = \begin{bmatrix} 2 & 3 \\ 3 & 4 \end{bmatrix}.

Chain Rule

Theorem 1 (Chain Rule for Multiple Variables). For composite functions f(g1(x),g2(x),…,gn(x))f(g_1(x), g_2(x), \dots, g_n(x)): dfdx=∑i=1n∂f∂gidgidx\frac{df}{dx} = \sum_{i=1}^n \frac{\partial f}{\partial g_i} \frac{dg_i}{dx}

Remark 2. The chain rule is fundamental for backpropagation in neural networks, allowing us to compute gradients through composite functions.

    """Analytical Hessian for f(x,y)"""
    # d^2f/dx^2 = 2, d^2f/dy^2 = 4, d^2f/dxdy = 3

    """Approximate the Hessian numerically using finite differences"""
    f_xx = (f(x+h, y) - 2*f(x, y) + f(x-h, y)) / (h**2)

Important Distributions in ML

    The Gaussian distribution, commonly known as the Normal distribution or the "Bell Curve," is arguably the most important concept in statistics. It describes a continuous probability distribution where most data points cluster around a central mean, and the rest taper off symmetrically toward the extremes.

    If only the mean and variance of a random variable is known to us, assuming a Gaussian imposes the least amount of additional structure or bias.     Standard Deviation (σ\sigma): This determines the spread.A small σ\sigma results in a tall, skinny curve (data is tightly packed).A large σ\sigma results in a short, wide curve (data is spread out).

    It models a single trial of an experiment that has exactly two possible outcomes: "Success" or "Failure."

    If a coin is flipped once, or a single lightbulb is checked to see if it’s broken, you are looking at a Bernoulli trial.

    In this distribution, we represent the outcomes with numbers to make them easier to calculate:
    1 (Success): Occurs with probability pp
    0 (Failure): Occurs with probability qq, which is simply 1−p1 - p
    Because there are only two data points, the math stays relatively lean:
    Mean (μ\mu): Simply pp. If a coin has a 70% chance of heads, the average outcome over time is 0.7.
    Variance (σ2\sigma^2): Calculated as p(1−p)p(1 - p).

    Most computer-generated random numbers (between 0 and 1) use a Uniform distribution as their base.

Bayes’ Theorem

Theorem 2 (Bayes’ Theorem). For events AA and BB with P(B)>0P(B) > 0: P(A|B)=P(B|A)P(A)P(B)P(A|B) = \frac{P(B|A)P(A)}{P(B)}

Bayes’ Theorem is the mathematical way of saying, "Changing your mind when you get new information."
Most probability is about predicting the future based on what we know now. Bayes’ Theorem flips that: it’s about looking at an outcome that already happened and updating our belief about what caused it. It is the foundation of Bayesian Statistics and is used in everything from email spam filters to medical diagnoses.
The theorem is built on the idea of Conditional Probability—the probability of an event AA occurring, given that event BB has already occurred P(A|B)P(A|B).
P(A|B)P(A|B) (The Posterior): What we want to know. The probability of AA being true given the new evidence BB.
P(B|A)P(B|A) (The Likelihood): If AA were true, how likely is it that we would see evidence BB?
P(A)P(A) (The Prior): Our "gut feeling" or existing knowledge before we saw the new evidence.
P(B)P(B) (The Evidence): The total probability of seeing evidence BB under all possible circumstances.

Example 8 (Medical Diagnosis). In microbiology, Bayes’ theorem can help calculate the probability of a disease given a test result: P(Disease|Test+)=P(Test+|Disease)P(Disease)P(Test+)P(\text{Disease}|\text{Test+}) = \frac{P(\text{Test+}|\text{Disease})P(\text{Disease})}{P(\text{Test+})}

Bayes’ Theorem is revolutionary because it allows for iterative learning.
You start with a belief (Prior).
You observe new data (Evidence).
You calculate a new, updated belief (Posterior).
That Posterior then becomes your new Prior for the next time you see data.

Maximum Likelihood Estimation (MLE)

If Bayes’ Theorem is about updating your beliefs based on new data, Maximum Likelihood Estimation (MLE) is about finding the "best fit" for your data without any prior assumptions.
In simple terms, MLE is a method used to estimate the parameters of a probability distribution (like the mean μ\mu or standard deviation σ\sigma) by picking the values that make your observed data most likely to have happened.
The Intuition: "Reverse Engineering"
Imagine you find a bag of 100 marbles. You pull out 10 marbles, and 9 of them are blue.

Case A: The bag is 10

Case B: The bag is 50

Case C: The bag is 90

MLE concludes that the "best" estimate for the percentage of blue marbles in the bag is 90%, because that value maximizes the probability of seeing the data you just collected.

To perform MLE, we define a Likelihood Function, denoted as L(θ)L(\theta).
Define the Likelihood: If we assume our data points (x1,x2,...xnx_1, x_2, ... x_n) are independent, the total likelihood is the product of their individual probabilities:L(θ)=P(x1|θ)⋅P(x2|θ)⋅...⋅P(xn|θ)L(\theta) = P(x_1|\theta) \cdot P(x_2|\theta) \cdot ... \cdot P(x_n|\theta)
The Log Trick: Multiplying many small probabilities results in tiny numbers that are hard to work with. Instead, we take the Natural Log (ln\ln) of the function. This turns the products into sums:ln⁡(L(θ))=∑ln⁡(P(xi|θ))\ln(L(\theta)) = \sum \ln(P(x_i|\theta))
Optimize: We use calculus (finding the derivative and setting it to zero) to find the exact point where the curve hits its highest peak. That peak is our Maximum Likelihood Estimate (θ̂\hat{\theta}).

Example 10 (MLE of a Coin Toss). Suppose we toss a biased coin nn times and get kk heads. Let pp be the probability of a head. The likelihood function is L(p)=pk(1−p)n−kL(p) = p^k(1-p)^{n-k}. Taking the log-likelihood: ln⁡L(p)=kln⁡(p)+(n−k)ln⁡(1−p)\ln L(p) = k\ln(p) + (n-k)\ln(1-p). Setting the derivative to zero: kp−n−k1−p=0⟹p̂=kn\frac{k}{p} - \frac{n-k}{1-p} = 0 \implies \hat{p} = \frac{k}{n}. This mathematical proof justifies why our intuitive guess (heads divided by total tosses) is the optimal estimate.

MLE in Common Distributions:
We have likely been using MLE results our whole life without realizing it!
Normal Distribution: If you have a set of heights, the MLE for the mean (μ\mu) is just the arithmetic average of the heights.
Bernoulli Distribution: If you flip a coin NN times and get HH heads, the MLE for the probability of heads (pp) is simply H/NH/N.
MLE vs. Bayesian Estimation
The main difference is how they treat the "truth":
MLE: Assumes the parameter (θ\theta) is a fixed but unknown constant. It only cares about the data you have right now.
Bayesian: Assumes the parameter is a random variable itself. It combines the data with a "Prior" (what you thought before you saw the data).
MLE is the "workhorse" of modern statistics and Machine Learning.
Logistic Regression uses MLE to find the best weights for classification.

Neural Networks essentially use a form of MLE (minimizing "Loss" is often mathematically equivalent to maximizing "Likelihood").

Optimization Theory

Gradient Descent

Definition 12 (Gradient Descent). An iterative optimization algorithm to find the minimum of a function: 𝒙(k+1)=𝒙(k)−η∇f(𝒙(k))\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} - \eta \nabla f(\mathbf{x}^{(k)}) where η>0\eta > 0 is the learning rate.

Example 11 (Step-by-Step Gradient Descent). Let’s minimize f(x)=(x−3)2+2f(x) = (x-3)^2 + 2. The gradient is f′(x)=2(x−3)f'(x) = 2(x-3). Assume learning rate η=0.1\eta = 0.1 and starting point x(0)=10x^{(0)} = 10.

  • Iteration 1: Calculate gradient: g(0)=2(10−3)=14g^{(0)} = 2(10-3) = 14.
    Update xx: x(1)=x(0)−ηg(0)=10−0.1(14)=8.6x^{(1)} = x^{(0)} - \eta g^{(0)} = 10 - 0.1(14) = 8.6.

  • Iteration 2: Calculate gradient: g(1)=2(8.6−3)=11.2g^{(1)} = 2(8.6-3) = 11.2.
    Update xx: x(2)=x(1)−ηg(1)=8.6−0.1(11.2)=7.48x^{(2)} = x^{(1)} - \eta g^{(1)} = 8.6 - 0.1(11.2) = 7.48.

With each step, xx gets closer to the analytical minimum of x=3x = 3.

Learning Rate and Convergence

Definition 13 (Learning Rate). The step size η\eta in gradient descent. Too large causes divergence; too small causes slow convergence.

Remark 3. In practice, adaptive learning rates (like Adam optimizer) are often used instead of fixed η\eta.

Stochastic Gradient Descent (SGD) and Momentum

Definition 14 (Stochastic Gradient Descent). A variant of gradient descent that uses a random subset (mini-batch) of data to compute gradient approximations: 𝒙(k+1)=𝒙(k)−η∇fi(𝒙(k))\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} - \eta \nabla f_i(\mathbf{x}^{(k)}) where fif_i is the loss for a single or small batch of data points.

Definition 15 (Gradient Descent with Momentum). Momentum accelerates gradient descent by considering the history of past gradients to smooth out updates. 𝒗(k+1)=γ𝒗(k)+η∇f(𝒙(k))\mathbf{v}^{(k+1)} = \gamma \mathbf{v}^{(k)} + \eta \nabla f(\mathbf{x}^{(k)}) 𝒙(k+1)=𝒙(k)−𝒗(k+1)\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} - \mathbf{v}^{(k+1)} where γ∈[0,1)\gamma \in [0,1) is the momentum parameter (typically 0.9). It acts similarly to a ball rolling down a hill, gaining speed in consistent directions.

Mathematical Foundations of Neural Networks

The Neuron Model

Definition 17 (Artificial Neuron). A mathematical model of a biological neuron: y=ϕ(∑i=1nwixi+b)=ϕ(𝒘T𝒙+b)y = \phi\left(\sum_{i=1}^n w_i x_i + b\right) = \phi(\mathbf{w}^T\mathbf{x} + b) where:

  • 𝒙∈ℝn\mathbf{x} \in \mathbb{R}^n: Input vector

  • 𝒘∈ℝn\mathbf{w} \in \mathbb{R}^n: Weight vector

  • b∈ℝb \in \mathbb{R}: Bias term

  • ϕ\phi: Activation function

Common Activation Functions

  • Sigmoid: σ(z)=11+e−z\sigma(z) = \frac{1}{1 + e^{-z}}

  • ReLU: ReLU(z)=max⁡(0,z)\text{ReLU}(z) = \max(0, z)

  • Hyperbolic Tangent: tanh⁡(z)=ez−e−zez+e−z\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}

Loss Functions

Definition 18 (Mean Squared Error (MSE)). For regression problems: L(𝒘)=1N∑i=1N(yi−ŷi)2L(\mathbf{w}) = \frac{1}{N} \sum_{i=1}^N (y_i - \hat{y}_i)^2 where ŷi\hat{y}_i is the predicted value.

Definition 19 (Cross-Entropy Loss). For classification problems: L(𝒘)=−1N∑i=1N∑c=1Cyi,clog⁡(ŷi,c)L(\mathbf{w}) = -\frac{1}{N} \sum_{i=1}^N \sum_{c=1}^C y_{i,c} \log(\hat{y}_{i,c}) where CC is the number of classes.

Backpropagation Mathematics

Theorem 3 (Backpropagation). Given a neural network with LL layers, the gradient of the loss with respect to weights in layer ll is: ∂L∂𝑾(l)=δ(l)(𝒂(l−1))T\frac{\partial L}{\partial \mathbf{W}^{(l)}} = \delta^{(l)} (\mathbf{a}^{(l-1)})^T where δ(l)\delta^{(l)} is the error term at layer ll, computed recursively: δ(l)={(𝒂(L)−𝒚)⊙ϕ′(z(L))for output layer(𝑾(l+1))Tδ(l+1)⊙ϕ′(z(l))for hidden layers\delta^{(l)} = \begin{cases} (\mathbf{a}^{(L)} - \mathbf{y}) \odot \phi'(z^{(L)}) & \text{for output layer} \\ (\mathbf{W}^{(l+1)})^T \delta^{(l+1)} \odot \phi'(z^{(l)}) & \text{for hidden layers} \end{cases} Here ⊙\odot denotes element-wise multiplication.

import numpy as np
        
        class SimpleNeuron:
        """A single neuron implementation with backpropagation"""
        
        def __init__(self, input_size):
        self.weights = np.random.randn(input_size) * 0.1
        self.bias = np.random.randn() * 0.1
        
        def sigmoid(self, x):
        """Sigmoid activation function"""
        return 1 / (1 + np.exp(-x))
        
        def sigmoid_derivative(self, x):
        """Derivative of sigmoid function"""
        s = self.sigmoid(x)
        return s * (1 - s)
        
        def forward(self, inputs):
        """Forward pass"""
        self.inputs = inputs
        self.z = np.dot(inputs, self.weights) + self.bias
        self.output = self.sigmoid(self.z)
        return self.output
        
        def backward(self, d_loss):
        """
        Backward pass
        d_loss: derivative of loss with respect to output
        """
        # Derivative through activation function
        d_z = d_loss * self.sigmoid_derivative(self.z)
        
        # Gradients for weights and bias
        d_weights = np.outer(self.inputs, d_z)
        d_bias = d_z
        
        # Gradient for inputs (for chaining in multi-layer networks)
        d_inputs = np.dot(d_z, self.weights.T)
        
        return d_weights, d_bias, d_inputs
        
        def update(self, d_weights, d_bias, learning_rate=0.1):
        """Update weights and bias using gradient descent"""
        self.weights -= learning_rate * d_weights
        self.bias -= learning_rate * d_bias
        
        # Example usage
        neuron = SimpleNeuron(input_size=3)
        
        # Forward pass
        inputs = np.array([0.5, 1.0, -0.5])
        output = neuron.forward(inputs)
        print(f"Input: {inputs}")
        print(f"Output: {output}")
        
        # Backward pass (simulating a loss gradient of 0.1)
        d_loss = 0.1
        d_weights, d_bias, d_inputs = neuron.backward(d_loss)
        
        print(f"\nGradient for weights: {d_weights}")
        print(f"Gradient for bias: {d_bias}")
        
        # Update parameters
        neuron.update(d_weights, d_bias, learning_rate=0.1)
        print(f"\nUpdated weights: {neuron.weights}")
        print(f"Updated bias: {neuron.bias}")
